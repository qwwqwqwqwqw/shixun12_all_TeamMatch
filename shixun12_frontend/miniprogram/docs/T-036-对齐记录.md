## 6. 未对齐/待确认项

- **字段命名风格**：前端 Mock 采用驼峰（`exitMode`, `exitDate`, `trustScore`, `requestType`），M4 真实接口是返回驼峰还是下划线（`exit_mode`, `exit_date`, `trust_score`, `request_type`）？需确认是否前端需加一层字段映射。
- **用户 ID 来源**：前端 Mock 固定 `currentUserId: 1` 和 Service 层 `CURRENT_USER_ID: 202`，实际运行时用户 ID 从哪里获取？登录态 Token 中解析还是单独的用户信息接口？
- **`project.in_progress` 状态**：展示 `...` 按钮是否需要判断项目阶段（recruiting 阶段不显示退出投票入口）？
- **发起投票接口**：队长点击 `...` 后跳转到投票发起页还是弹窗？请求体和返回体格式待 M4 定义。
- **退出成员是否仍在列表中**：当前 Mock 显示已退出成员（`status: 'exited'`）仍然在列表中，仅展示灰色标签和退出原因。M4 确认接口是否同样返回已退出成员？
- **`avatar` 空值处理**：Mock 中头像为空字符串，前端回退显示 `/images/default-avatar.png`。M4 确认是否返回完整 URL 或为空时需要前端处理。
- **`trustScore` 字段名**：`team_request` 中称为 `trustScore`，`team_member` 中也是 `trustScore`，是否统一字段名？
- **分页**：当前 Mock 返回全量数据，M4 的列表接口是否需要分页参数（`page`, `pageSize`）？
