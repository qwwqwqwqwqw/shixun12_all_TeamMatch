# M4 阶段开发与测试证据包

## 1. 概述

M4 模块负责 TeamMatch 平台的项目管理（发布/状态流转）、组队请求（邀请/申请/接受/拒绝/取消）、成员管理、退出流程（主动退出/投票退出）、信誉分计算规则等核心业务。本证据包汇总主链路功能、任务对应关系、单元测试结果、演示数据及设计文档，供验收与归档使用。

## 4. 项目状态机

### 4.1 Project 状态流转

```
recruiting（招募中）
    │  startProject（T-117，队长操作，失效所有 pending 请求）
    ▼
in_progress（进行中）
    │  endProject（T-118，队长操作，写 endedAt / evalDeadline）
    ▼
ended（已结束，互评窗口开放）
    │  evalDeadline 到期（懒检查 T-119）
    ▼
eval_closed（互评已关闭）
```

### 4.2 TeamRequest 状态流转

```
pending（待处理）
    ├── acceptRequest → accepted（T-124，自动写入 team_member）
    ├── rejectRequest → rejected（T-129）
    ├── cancelRequest → cancelled（T-130，发送方操作）
    └── startProject  → expired（项目开始时批量失效）
```

### 4.3 ExitVote 状态流转

```
voting（投票中，+24h 截止）
    │  closeVote（T-140，队长操作，事务+幂等）
    ▼
closed
    ├── result=pass  → team_member.status=exited，写信誉扣分流水
    └── result=reject → 目标成员继续留队
```
